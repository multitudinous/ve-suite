The code is rough, so the operations have to be very specific for everything to work properly.

- Start by making a file (say locations.dat) in the working directory for the application

- Add a block 14 to the parameter file with the view points file name on the next line
(the vrxpr.param.example should be good now)

-Start the app running and when a chosen point has been found, select Add View Point
on the View Points tab. Continue this until you have found all of the view points you would like
to store, then click Write Points to File.

- Currently the GUI isn't set up to dynamically display new points when read in from the file,
so in order to see your stored points in the selection box you will have to restart the app.

- At this point you should be in good shape to move around to the different points.
To do this simply select the desired point in the selection box and click the Move button.

- A major flaw comes in with loading new points into a file with existing points. Right now,
if you have file with existing points and find new points that you want to store, you will have to
manually copy the previously stored locations out of the file before writing new points to the file,
because the old points will be overwritten. After writing the new points to the file, you can go
back in and manually copy the previously stored points back into the file.
