Creating a parameter file for VE-Xplorer

In VE-Xplorer directory, open vrxpr.param. Here is a sample parameter file.

The parameter file is divided into blocks. The first line indicates how many blocks are to be loaded. In the sample parameter file, the first line indicates that 7 blocks will be loaded. This integer must be included and must be accurate. The 7 blocks in the sample parameter file are currently the only types of blocks that may be defined. No block is required and they may be inserted in any order. The user chooses which blocks to include based on the desired VE-Xplorer display.

The first and second blocks (0 and 1) allow the user to tweak the appearance of the display in the virtual environment.

Block 0: Must contain 3 lines:
--Scale values (must be greater than 0 to be visible)
--Translation values
--Rotation values

Block 1: Must contain 3 lines:
--scalarBar Position
-- scalarBar Z-Rotation
-- scalarBar Height and Width)

The third block (5) contains PIV data. This block is currently being developed and may change. Must contain 3 lines:
--Name of image file
--Location of lower left corner
--Orientation

The fourth block (8) contains vtk datasets such as unstructured grids or polydata such as surfaces or particle tracks. Must contain 6 lines:
--Scale
--Translation
--Rotation
--flowdata.vtk (vtkDataSet name)
--precomputed data slice directory. Insert JUNK or other undefined directory name if not applicable
--precomputed surface directory. Insert JUNK or other undefined directory name if not applicable

The fifth block (9) contains the geometry file. It can only contain files that can be visualized with perfly (e.g., *.stl, *.iv, *.pfb, *.obj). Must contain 6 lines:
--Transparency toggle (1 makes geometry transparent when visualizations are active)
-- Color flag (and 3 color values if flag = 1)
--Scale values for geometry file
--Translation values for geometry file
--Rotation values for geometry file
--Geometry file name

The sixth block (10) is the transient data loader. Must contain 18 lines:
--Number of directories containing vtk data to follow
--Data dcs scale values
--Data dcs translation values
--Data dcs rotation values
--Directory of vtk files
--Button ID: 0 = 3D_mesh, 1 = x-planes, 2 = y-planes, 3 = z-planes, 4 = particle cloud
--Directory of vtk files
--Button ID: 0 = 3D_mesh, 1 = x-planes, 2 = y-planes, 3 = z-planes, 4 = particle cloud
--Directory of vtk files
--Button ID: 0 = 3D_mesh, 1 = x-planes, 2 = y-planes, 3 = z-planes, 4 = particle cloud
--Directory of vtk files
--Button ID: 0 = 3D_mesh, 1 = x-planes, 2 = y-planes, 3 = z-planes, 4 = particle cloud
--Transient geometry data directory
--Transient geometry dcs scale values
--Transient geometry dcs translation values
--Transient geometry dcs rotation values
--Transient geometry transparency setting, color flag (and 3 color values if flag = 1)
--Duration of the transient sequence in seconds

The seventh block (11) contains the sound file. You must have Sound API working (OpenAL is recommended). Must contain 10 lines:
--Ambient background noise (0 = off, 1 = on)
--Retriggerable, prevents sound from being retriggered in the program (0 = off, 1 = on)
--Volume (range is 0 [soft] to 1 [loud])
--Cutoff
--Sound position x
--Sound position y
--Sound position z
--Sound file name
--Sound name alias (give a unique name of alphanumeric characters)

The final 3 blocks of 1 line each are mandatory:
--Warped contour scale value
--Navigation step file (increase to go faster)
--Streamline diameter, set to 0.0 if auto-value is sufficient (will eventually disappear from the parameter file and be put in the GUI)

If steamline diameter is set to 0, the diameter will be auto-scaled by VE-Xplorer. If it is greater than 0, it represents the actual diameter in feet. The relative scale value--bigger/smaller scales.
