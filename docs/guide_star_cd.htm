Using VE-Builder with StarCD

  1. Launch StarCD

  2. Ensure that a *.vrt and *.cel file is written of the region of interest. This region can either be a complete CFD model or a subsection of the model.

  3. Analyze data

  4. Post-processing > Load data

    1. In "File(s)" tab, click "open post file"

    2. In "Data" tab:

      Scalar Data > Turb Kinetic Energy

      Vector Data > Velocity Components UVW


       

  5. Click "Get Data"

  6. Click "View Post Registers"

  7. To add scalars:

    1. Click "Operate"

    2. Function > Load vertex data > turbulent energy (Registers 1-3 must contain vector data. Registers 4-6 must contain vertex data.)

    3. Select register

    4. Note: Although it is necessary to use vertex data rather than cell data, using vertex data might return an error. To solve this problem, copy the previous command line and add the register number, e.g. "enter oper, getv, te, 5, "

    5. Ensure that "using relative values" is selected

    6. Click "Apply" to put data in an empty register

    7. Click "Update list"

    8. Repeat steps 7.1-7.6 to add data to registers 5 and 6 (pressure and viscosity)

  8. The data is now stored.

  9. In pro-STAR window:

    1. Post > save user data

    2. Registers: all

    3. File type: coded

    4. Range: all vertices

    5. Click "Apply"

    6. Click "Close"

  10. Manually close file (enter "close *.usr")

  11. File > save as coded

  12. Unclick boundary file

  13. Click "Apply"

  14. Note: Checked files are written out

  15. Enter "close all"

You now have the geometry information and data. It is time to translate.
