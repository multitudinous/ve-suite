Gasifier Module Buttons

Graphical User Interface (UI)

The Gasifier Module's graphical user interface is divided into four folders. The third of these, Conditions, allows the user to specify whether the module models a one or two stage type gasifier. Also within this folder, the user may specify the Pressure Drop across the module.

The Gasifier module can be used to model either one stage or two stage gasifiers. One stage gasifiers are typified by a single set of injectors positioned in either a down flow (shown below) or upflow design. Two stage gasifiers usually have two larger compartments separated by a thin necked center section. Two stage gasifiers use two sets of injectors fired directly into each of the larger compartments (shown at right).

When the one stage type is specified, only the first injector of the User Interface is active. When a two stage gasifier is selected, the user must input values for each of the three injectors (see diagram at right).

The first folder, Flows, allows the user to specify the injector properties. The User Interface is organized such that each of the three available injectors is configured separately, toggled by the Fuel Injectors-level input field. Required fields include steam Flowrate as well as slurry properties including Temperature, Flowrate, and Percent Coal. The Re-Cycled Char is an externally computed value specified by the user to represent the percent char returning to the gasifier (typically from a feedback loop running through a Cyclone unit).

The Percent Water field is automatically calculated from the user specified field Percent Coal.

The Coal folder prompts for information regarding the chemical properties of the coal in the slurry. The workbench provides 5 preprogrammed coal types: Pittsburg #8, Illinois #5, Illinois #6, Petcoke, and Pocahantas #3.

The Execution folder controls not only the number of iterations the module performs, but also aids in creating 3D data representations. The user must specify a Working Directory so that the module can then store results for later use by the VTK Viewer.

Summary Data (SUM)

The Summary box gives details of module outcomes including exit properties, module efficiencies, and the burnout percent.
